概览¶

版本控制¶

写过论文报告的人都有不断修改文件的经验，有时还需要保留不同版本便于对比替换，这就是最简单的版本控制了。通过手动实现的少量文件的版本控制还算可行，当文件数量达到以千计万计时就会显得十分吃力，这时就需要版本控制系统帮助了。

版本控制系统（ Version Control System），能够记录文件变化，进行版本回退和细节比较。

  • 本地版本控制系统

    它的工作原理，是在硬盘上保存补丁集（补丁是指文件修订前后的变化），通过应用相应的补丁，可以重新计算出各个版本的文件内容。其中最流行的一种叫做 RCS（Revision Control System），现今许多计算机系统上都还看得到它的踪影。甚至在流行的 Mac OS 系统上安装了开发者工具包之后，也可以使用 rcs 命令。

  • 集中式版本控制系统

  • 分布式版本控制系统

协同开发¶

通过上面所讲的集中式或分布式版本控制系统，能够依托网络实现线上协同开发，而非通过复制拷贝文件的协同。

本教程主要介绍其中一种布式版本控制系统 Git，以及使用 Git 做为唯一版本库格式的项目托管平台 Github。了解 Git 的工作原理，能更好地帮助我们在遇到问题时能查找原因。

说明 Git 是推荐掌握的技能，若理解和学习 Git 有一定难度，掌握 Github 的使用即可。
